# core rules（常時ロード）

このファイルは CLAUDE.md から `@` import され、常に文脈に載る（§4-3）。自己完結させる。

## 知識の置き場所
- 恒久・横断の知識は `docs/`（決定→`docs/decisions/`、設計→`docs/design/`）。**個人メモリに逃がさない**。
- コード・スキーマ・Issue を見れば分かる事実は decisions/ に書かない（§3-3）。

## Git / コミット規律
- `main` へ直接コミットしない。作業ブランチ `feat/<topic>` を切る。
- コミット/プッシュは原則ユーザーの指示後。コミットメッセージは命令形・要点1行＋必要なら箇条書き。

## 変更の作法
- 既存コードの近傍に合わせる（命名・コメント密度・イディオム）。
- リファクタにはテストを同伴させる（挙動不変は「変わっていない」をテストでしか担保できない / §5-6）。
- 新しい区分値（enum/union）を足したら、分岐の網羅を確認（書き忘れは静的検査で塞ぐ）。

## ドキュメント同期
- 決定が出たら `docs/decisions/NNNN-*.md` を追加 → `npm run gen:decisions` で索引を再生成（手編集しない）。
- 設計のずれは `docs/design/*.md` を直す。

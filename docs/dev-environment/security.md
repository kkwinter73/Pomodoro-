# security — 設定の責任分界・秘密情報の扱い

ハーネスの「層5: 配布と同期」のうち設定の責任分界（§7-2）。

## 設定ファイルの役割分担

| ファイル | 役割 | コミット |
|---|---|---|
| `.claude/settings.json` | チーム共有のハーネス設定（hook 配線・権限） | する（レビュー経路に乗せる） |
| `.claude/settings.local.json` | 個人 override | **しない**（`.gitignore` 済み） |

横断の標準は**コミットされる共有設定**に置き、レビューを通してドリフトを防ぐ。
個人都合の調整は local 側に閉じ込める。

## 秘密情報

- API キー・トークン等は**どのレイヤにも平文で書かない**。`.env`（gitignore 済み）で扱い、`.env.example` に項目だけ置く。
- 権限プロンプトで秘密を含むコマンドを許可するとき、**"Always allow" を選ばない**（settings に平文で刻まれる）。**"Allow once"** を使う。
- このポモドーロアプリは現状サーバ秘密を持たない（ADR 0001）。将来サーバ機能を足す場合はこの原則を最初から適用する。

## ガードの環境差耐性

hook はツール未インストール・環境変数未設定などの異常系では**無音で通す**（§5-5）。
ガードが環境差で作業を止めると信頼されず外されるため、「迷ったら邪魔をしない」を既定にしている。
詳細は [hooks-guide.md](hooks-guide.md)。
